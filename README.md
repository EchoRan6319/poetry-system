## 一、环境准备

### 1.1 安装 Node.js

Node.js 是运行本系统的基础环境，必须安装。

#### 步骤 1：下载安装包

1. 打开浏览器，访问：https://nodejs.org/
2. 点击 **"LTS"**（长期支持版）下载按钮
3. 下载完成后，双击运行安装程序

#### 步骤 2：安装 Node.js

1. 双击下载的安装包（如 `node-v20.x.x-x64.msi`）
2. 点击 **"Next"** 进入安装向导
3. 勾选 **"I accept the terms in the License Agreement"**，点击 **"Next"**
4. 选择安装路径（建议保持默认 `C:\Program Files\nodejs\`），点击 **"Next"**
5. 保持默认选项，点击 **"Next"**
6. 点击 **"Install"** 开始安装
7. 安装完成后点击 **"Finish"**

#### 步骤 3：验证安装

1. 按 `Win + R` 键，输入 `cmd`，按回车打开命令提示符
2. 输入以下命令，按回车：
   ```cmd
   node --version
   ```
3. 如果显示版本号（如 `v20.10.0`），说明安装成功
4. 再输入：
   ```cmd
   npm --version
   ```
5. 如果显示版本号（如 `10.2.3`），说明 npm 也安装成功

> ⚠️ **注意**：如果提示 "'node' 不是内部或外部命令"，请跳到 [5.1 环境变量问题](#51-node-命令无法识别)

## 二、项目部署

### 2.1 获取项目文件

项目文件通常存储在 U 盘或共享文件夹中，包含以下文件：

```
poetry-system/                 # 根目录
├── poetry-system/             # 项目目录（重要：在此目录下运行命令）
│   ├── node_modules/          # 依赖包文件夹（如有）
│   ├── public/                # 静态资源
│   │   └── 题库.json          # 题目数据
│   ├── src/                   # 源代码
│   │   ├── views/             # 页面组件
│   │   ├── utils/             # 工具函数
│   │   └── ...
│   ├── index.html             # 入口页面
│   ├── package.json           # 项目配置
│   └── vite.config.js         # 构建配置
├── README.md                  # 本文件
└── ...
```

### 2.2 进入项目目录

**重要：项目代码在子目录中，需要先进入子目录**

1. 打开文件资源管理器，进入根文件夹 `poetry-system`
2. 在地址栏输入 `cmd`，按回车（在当前目录打开命令提示符）
3. 进入项目子目录：
   ```cmd
   cd poetry-system
   ```

### 2.3 安装项目依赖

1. 在项目子目录中，输入以下命令安装依赖：
   ```cmd
   npm install
   ```
2. 等待安装完成（可能需要 1-3 分钟，取决于网络速度）
3. 安装完成后，会生成或更新 `node_modules` 文件夹

> ⚠️ **注意**：如果安装过程中出现错误，请跳到 [5.2 依赖安装问题](#52-依赖安装失败)

## 三、启动系统

### 3.1 启动开发服务器

1. 确保已在项目子目录 `poetry-system` 中（命令提示符路径应显示 `...\poetry-system\poetry-system>`）
2. 输入以下命令启动系统：
   ```cmd
   npm run dev
   ```
3. 等待启动成功，会显示以下信息：
   ```
   VITE v5.x.x  ready in xxx ms

   ➜  Local:   http://localhost:5173/
   ➜  Network: use --host to expose
   ➜  press h + enter to show help
   ```

### 3.2 访问系统

系统启动后，有两个主要页面：

| 页面 | 地址 | 用途 |
|------|------|------|
| **主页面（大屏）** | http://localhost:5173/screen | 投屏展示，供选手和观众观看 |
| **控制台** | http://localhost:5173/admin | 主持人操作，控制比赛流程 |

**操作步骤**：

1. 打开浏览器（推荐使用 Chrome 或 Edge）
2. 在地址栏输入 `http://localhost:5173/screen`，按回车
3. 按 `F11` 键全屏显示
4. 再打开一个浏览器窗口，输入 `http://localhost:5173/admin`
5. 将控制台窗口拖到主持人操作电脑上

> 💡 **提示**：主页面和控制台可以通过快捷键 `Ctrl + R` 或 `F5` 刷新

## 四、使用说明

### 4.1 比赛流程

```
开场 → Emoji猜诗 → 宫格寻诗 → 易错找茬 → 中原风华 → 经典填词 → 飞花令 → 颁奖
```

### 4.2 控制台操作

#### 页面布局

控制台分为几个区域：

1. **顶部状态栏**：显示连接状态、当前环节
2. **左侧题目列表**：选择要展示的题目
3. **中间控制区**：
   - 开始/暂停计时
   - 显示/隐藏答案
   - 下一题
4. **右侧计分板**：
   - 各学院当前分数
   - 加分/减分按钮

#### 基本操作

| 操作 | 步骤 |
|------|------|
| **展示题目** | 点击左侧题目列表中的题目 |
| **开始计时** | 点击"开始计时"按钮 |
| **显示答案** | 点击"显示答案"按钮 |
| **加分** | 点击对应学院右侧的 "+10" 或 "+20" |
| **减分** | 点击对应学院右侧的 "-10" |
| **下一题** | 点击"下一题"按钮或选择新题目 |

#### 快捷键

| 快捷键 | 功能 |
|--------|------|
| `空格键` | 开始/暂停计时 |
| `Enter` | 显示答案 |
| `→` 或 `↓` | 下一题 |
| `←` 或 `↑` | 上一题 |

### 4.3 大屏显示

大屏会自动同步控制台的操作：

- 展示题目时显示题目内容
- 计时器倒计时
- 答案揭晓时显示正确答案
- 底部显示各学院实时分数

## 五、常见问题

### 5.1 Node 命令无法识别

**现象**：输入 `node --version` 提示 "'node' 不是内部或外部命令"

**原因**：Node.js 环境变量未配置

**解决方案**：

#### 方法一：使用完整路径运行

1. 找到 Node.js 安装路径（默认 `C:\Program Files\nodejs\`）
2. 使用完整路径运行命令：
   ```cmd
   "C:\Program Files\nodejs\node.exe" --version
   ```
3. 启动项目时也使用完整路径：
   ```cmd
   "C:\Program Files\nodejs\node.exe" "C:\Program Files\nodejs\node_modules\npm\bin\npm-cli.js" run dev
   ```

#### 方法二：配置环境变量（推荐）

1. 右键"此电脑" → 属性 → 高级系统设置
2. 点击"环境变量"按钮
3. 在"系统变量"中找到 `Path`，双击编辑
4. 点击"新建"，添加 Node.js 安装路径：
   ```
   C:\Program Files\nodejs\
   ```
5. 点击"确定"保存
6. 重新打开命令提示符，再次尝试 `node --version`

---

### 5.2 依赖安装失败

**现象**：运行 `npm install` 时报错

**可能原因及解决方案**：

#### 原因 1：网络问题

**解决方案**：使用淘宝镜像
```cmd
npm config set registry https://registry.npmmirror.com
npm install
```

#### 原因 2：权限问题

**解决方案**：以管理员身份运行命令提示符
1. 在开始菜单搜索 "cmd"
2. 右键"命令提示符" → "以管理员身份运行"
3. 进入项目目录，再次运行 `npm install`

#### 原因 3：node_modules 损坏

**解决方案**：删除后重新安装
```cmd
rmdir /s /q node_modules
npm install
```

#### 原因 4：不在正确的目录

**现象**：提示 "Missing script: dev" 或找不到 package.json

**解决方案**：确保已进入项目子目录
```cmd
cd poetry-system
npm install
npm run dev
```

---

### 5.3 端口被占用

**现象**：启动时提示 "Port 5173 is already in use"

**解决方案**：

#### 方法一：关闭占用端口的程序

1. 查找占用 5173 端口的进程：
   ```cmd
   netstat -ano | findstr :5173
   ```
2. 记录最后一列的 PID 数字
3. 结束进程：
   ```cmd
   taskkill /PID <PID> /F
   ```

#### 方法二：更换端口启动

```cmd
npm run dev -- --port 3000
```
然后访问 `http://localhost:3000/screen`

---

### 5.4 页面显示异常

**现象**：页面空白、样式错乱或数据不更新

**解决方案**：

1. **刷新页面**：按 `Ctrl + R` 或 `F5`
2. **清除缓存**：按 `Ctrl + Shift + R` 强制刷新
3. **清除 LocalStorage**：
   - 按 `F12` 打开开发者工具
   - 切换到 "Application"（应用）标签
   - 左侧选择 "Local Storage" → `http://localhost:5173`
   - 右键点击 `poetry-competition-state`，选择 "Delete"
   - 刷新页面

---

### 5.5 控制台和大屏不同步

**现象**：控制台操作后，大屏没有更新

**解决方案**：

1. 检查两个页面是否都正常打开
2. 刷新大屏页面（`F5`）
3. 如果仍不同步，尝试：
   - 关闭所有浏览器窗口
   - 重新访问 `http://localhost:5173/screen`
   - 重新访问 `http://localhost:5173/admin`

---

### 5.6 npm 脚本无法运行（PowerShell 限制）

**现象**：在 PowerShell 中运行 `npm run dev` 报错 "无法加载脚本"

**原因**：PowerShell 执行策略限制

**解决方案**：

#### 方法一：使用命令提示符（推荐）

不使用 PowerShell，改用 cmd：
1. 按 `Win + R`，输入 `cmd`，回车
2. 进入项目目录，运行 `npm run dev`

#### 方法二：修改 PowerShell 执行策略

1. 以管理员身份打开 PowerShell
2. 输入：
   ```powershell
   Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
   ```
3. 输入 `Y` 确认
4. 重新打开 PowerShell，再次尝试

## 六、应急处理

### 6.1 系统崩溃恢复

如果系统意外关闭，按以下步骤恢复：

1. **重新启动服务器**：
   ```cmd
   npm run dev
   ```

2. **恢复比赛状态**：
   - 系统会自动从 LocalStorage 恢复
   - 包括当前题目、分数、计时器等

3. **如果恢复失败**：
   - 清除 LocalStorage（见 5.4）
   - 手动重新选择题目
   - 手动调整分数

### 6.2 备用方案

如果 Node.js 环境完全无法运行，可以使用备用方案：

1. **静态页面模式**：
   - 打开 `index.html` 文件
   - 可以查看题目，但无法使用计时和计分功能

2. **手动计分**：
   - 准备纸质计分表
   - 主持人手动记录分数

### 6.3 联系技术支持

如果遇到无法解决的问题：

1. 记录错误信息（截图或拍照）
2. 记录操作步骤
3. 联系技术人员
